Commits frequentes e bem descritos são importantes porque:

Facilitam entender as mudanças no projeto
Ajudam a identificar e corrigir erros rapidamente
Melhoram a organização e o trabalho em equipe
2.
Restaurar arquivo pelo History: usado quando o erro está em um arquivo específico
Reverter commit (Revert): usado quando é necessário desfazer todas as mudanças de um commit
3.

O histórico de commits permite visualizar todas as alterações, ajudando a identificar em qual momento o erro surgiu e entender o que causou o bug.
